Translated Template for theses for Chemnitz University of Technology
======================================================================

**This is a translated version of the german template.**

Every degree course and every subsequent academic qualification usually ends 
with the submission of a scientific thesis. Especially in mathematics,
the natural sciences and computer science, the use of the text typesetting 
system LaTeX.

This project provides a template for scientific theses at the TU Chemnitz.
It is intended to provide you with a toolbox of proven LaTeX constructs, so
that you can concentrate on the content of your thesis.

Furthermore, this template should contribute to the standardisation of the
layout of theses. However, it does not claim to be the only recognised template
for theses. The specifications of the various fields differ too much for that.


Status of the project

at present. However, the template is already in a that could be quite helpful.

Design principles
------------------

The template does not claim to be a flexibly configurable macro package.
Instead, we regard it as a collection of frequently used code suggestions.
This guarantees that the template can be used with knowledge from standard
LaTeX textbooks and does not require any individual training effort. In
addition, the complexity of the template remains which, in our view, is
particularly beneficial for LaTeX beginners. Own macros only where they are
indispensable (e.g. for formatting the title page, etc.).

In addition to adapting the layout, the template guide should also provide a
number of helpful tips on good text typesetting, wording advice, correction
tips, dos & don'ts, etc.


Join
---------

We appreciate your support. We are happy to receive suggestions for
suggestions. This does not necessarily have to be LaTeX code. Anything can be
count as a contribution ...

* Bug reports: These are to be reported as issues in GitLab.
* Supplements and individual specifications from scientific
  scientific societies, faculties or individual professorships: If possible,
  please also report these as a GitLab issue.
* Supplementary advice, tips \& tricks, etc.: This should also be reported as
  a GitLab-issue.
* Code suggestions can be submitted as a separate Git branch. Once
  the work is ready for our review, a merge request can be submitted in GitLab.
  A merge request can be submitted, which we will then review.

The above list is not exhaustive. We are happy to receive all kinds of hints.

We reserve the right to adapt proposals to a uniform style guide and the design
principles described above.
